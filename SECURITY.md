# Security Policy

## Scope

OllamAGI is a **local-first** tool designed to run on your own machine. It is not a multi-tenant service. The security model assumes a single trusted user with local access.

## What OllamAGI can do

OllamAGI agents run with significant privileges by design:

- Spawn Docker containers with bind-mounts to your home directory
- SSH back to the host from inside containers
- Execute arbitrary code generated by an LLM
- Access the Docker socket (`/var/run/docker.sock`)

**Do not expose port 7654 to the public internet without authentication.** Use SSH tunnelling (as documented in the README) for remote access.

## Responsible Use

- The `security` flow type and Pentester agent are for **authorized testing only**
- Only run security flows against targets you own or have explicit written permission to test
- OllamAGI does not include any built-in rate limiting or access controls

## Reporting a Vulnerability

If you discover a security issue:

1. **Do not open a public GitHub issue**
2. Email: yandesbiens420@gmail.com
3. Include: description, reproduction steps, potential impact
4. Allow up to 72 hours for an initial response

We will work with you to understand and address the issue before public disclosure.

## Known Risks

| Risk | Mitigation |
|---|---|
| LLM generates malicious code | Review agent output in the dashboard before relying on results |
| Docker socket exposure | Limit who can access port 7654 |
| SSH key compromise | Use a dedicated key with limited authorized_keys scope |
| Prompt injection via web-scraped content | Agents run in isolated containers; review outputs |
