# OllamAGI — CODEX

Developer reference. Updated as the system evolves.

## What this is

OllamAGI is a local-first autonomous agent platform. It takes a natural-language objective, decomposes it into tasks and subtasks using Ollama LLMs, executes code in isolated Docker containers, auto-fixes failures, and feeds all results back into Hermes — a persistent SQLite cognitive memory system.

It is a daily-driver tool, not a demo. All features described here are working and in production use.

## Key design decisions

### Immediate flow stop
`FlowStoppedException(BaseException)` — subclasses BaseException so it bypasses all `except Exception` handlers. LLM calls run in daemon threads, main thread polls a `queue.Queue` every 0.5s. Stop propagates within ~500ms. Docker containers are killed immediately via `docker kill`.

### Python execution in containers
Code is injected via `put_archive` (tar) and run with `python3 /tmp/ollamagi_task.py` directly — never via bash heredocs (bash chokes on triple-quotes, `$vars`, emojis, etc.). `exec_python()` in `docker_manager.py`.

### No Rust compilation in containers
Agents use `pip install --prefer-binary` to avoid source builds. Pydantic v2 and polars are explicitly banned in the code prompt (they require Rust/cargo). The agent prompt lists safe alternatives.

### Retry + interruptible sleep
`interruptible_sleep(seconds, flow_id)` sleeps in 0.5s chunks, checking the stop event. Used in retry delays between LLM call failures so Stop works immediately even during waits.

### Token persistence
`_tokens` dict is embedded in `flow.json` on every `_save()`. Survives server restarts. `/api/tokens` aggregates in-memory (session) + disk (all-time) + returns `_session`, `_alltime`, `_flows` keys.

### Config
Everything in `core/config.py` reads from env vars. `HOME_DIR` in docker_manager uses `OLLAMAGI_USER_HOME` env var (defaults to `Path.home()`). Model names are env-configurable with public Ollama Hub defaults.

## Agent roles

| Role key | Purpose | Model key |
|---|---|---|
| `primary_agent` | Main orchestration + synthesis | orchestrator |
| `generator` | Subtask plan creation | orchestrator |
| `refiner` | Plan review | orchestrator |
| `coder` | Python/bash code writing | coder |
| `installer` | System setup, DevOps | coder |
| `researcher` | Information gathering | orchestrator |
| `adviser` | Strategic guidance | orchestrator |
| `reflector` | Failure analysis | orchestrator |
| `architect` | System design | orchestrator |
| `monetizer` | ROI analysis | orchestrator |
| `pentester` | Security (authorized only) | tools |

Unknown role names fall back to `primary_agent`. The `planning_agent` and `research_agent` names sometimes generated by LLMs are not in `ALL_ROLES` — they silently fall back.

## Container strategy

| Type | Image | Used for |
|---|---|---|
| `python` | python:latest | coder / installer agents |
| `generic` | debian:latest | bash tasks, planning agents |
| `pentest` | vxcontrol/kali-linux:latest | pentester agent |

Each subtask gets a **fresh container**. Bootstrap (`_bootstrap_python`) runs detached (non-blocking) and installs python3 + pip + common packages + uv. For debian/generic, it first installs python3 via apt-get since the base image doesn't have it.

Containers have:
- `/work` → `workspace/<flow-id>/` on host (bind-mounted)
- Host home dir bind-mounted
- Docker socket for DinD
- SSH key injected at `/root/.ssh/id_ed25519`

## Hermes memory integration

Hermes is optional. If `HERMES_DB` doesn't point to a real file, all memory calls are silently no-ops (wrapped in try/except in `memory_bridge.py`).

When available:
- `context_for_task(description)` — semantic search, returns relevant beliefs/memories as context string injected into every subtask
- `_memory_distill(flow_id, task_title, result)` — after each task, LLM extracts 1-3 beliefs and stores them
- `extract_and_store()` — bulk extraction at flow end

Hermes DB path: `~/.hermes/cognitive_memory.sqlite` (configurable via `HERMES_DB` env var).

## Known issues / active work

| Issue | Status | Notes |
|---|---|---|
| LLM-generated agent names (planning_agent, research_agent) not in ALL_ROLES | Known | Falls back to primary_agent, works but non-ideal. Add these aliases or improve the role generation prompt |
| Debian containers missing python3 | Fixed | `_bootstrap_python` now installs python3 via apt-get first |
| Bash scripts assuming python3 is available | Fixed | Prompt now says to install via apt-get if needed |
| Pydantic v2 / Rust compilation in containers | Fixed | `--prefer-binary` flag + explicit ban in code prompt |
| Ollama 500 errors leave flow stuck as "running" | Fixed | Outer `except Exception` in `run_flow` now marks flow as `failed`; startup cleanup marks stale "running" flows as "stopped" |
| Ollama timeout | Fixed | Increased to 600s (10 min); `_generate_tasks` and `_generate_subtasks` retry up to 3× with interruptible 5s sleep |
